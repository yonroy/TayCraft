Bài G4

LSTM — một ô nhớ — Long Short-Term Memory

Họ tên: Ngày: Thời gian: ~14 phút
Vì sao quan trọng. RNN trơn quên nhanh (gradient teo). LSTM cho đường ô nhớ c gần như tuyến tính (cổng quên ≈ 1 → giữ nguyên) → gradient chảy xa không teo, nhớ phụ thuộc dài hạn. Thống trị NLP/seq trước Transformer.
f, i, o = σ(…) (cổng quên/vào/ra) · c̃ = tanh(…) (ứng viên) · ct = f⊙ct−1 + i⊙c̃ · ht = o⊙tanh(ct)
💡 Trực giác Ô nhớ c như băng ghi nhớ chạy thẳng. Cổng quên f quyết bôi bao nhiêu băng cũ; cổng vào i quyết ghi thêm bao nhiêu cái mới; cổng ra o quyết đọc ra bao nhiêu cho trạng thái h. f≈1, i≈0 → giữ trí nhớ nguyên vẹn qua nhiều bước.

LSTM — một ô nhớ — lời giải

1

Cập nhật ô nhớ

ct = 0.28 + 0.30 = 0.58
2

Trạng thái ẩn

ht = 0.8·tanh(0.58) = 0.8·0.52 = 0.42
▦

Băng ô nhớ c chạy ngang + 3 cổng

f · i · o
Băng c chạy ngang gần như tuyến tính (f điều tiết giữ/bôi). LSTM tách c (nhớ) và h (đầu ra), 3 cổng; GRU gộp lại, 2 cổng → gọn hơn (Bài G5).
✎ Tự kiểm tra — đáp án
  1. f ≈ 1? → Giữ gần như nguyên ô nhớ cũ → nhớ lâu.
  2. Vì sao chống vanishing? → Đường ô nhớ c gần tuyến tính → gradient chảy xa không teo.
Rút ra. LSTM = ô nhớ c + 3 cổng; f≈1 giữ trí nhớ dài, chống vanishing. Bài tiếp G5: GRU — phiên bản gọn 2 cổng.
