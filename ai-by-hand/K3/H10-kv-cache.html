KV Cache

— sinh token tiếp theo tái dùng K,V cũ · chỉ tính token mới · điền ô trắng
Vì sao quan trọng. Khi LLM sinh từng token, mỗi bước token mới phải chú ý tới tất cả token trước. Tính lại K,V của mọi token cũ ở mỗi bước thì cực phí. KV cache lưu K,V đã tính; bước mới chỉ tính k,v cho token mới rồi nối vào cache, và làm một hàng attention cho query mới.
