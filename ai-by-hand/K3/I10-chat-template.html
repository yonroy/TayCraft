Chat Template — gói hội thoại bằng token đặc biệt

Họ tên: Ngày: Thời gian: ~11 phút
Vì sao quan trọng. Mô hình chỉ thấy một chuỗi token phẳng. Chat template thêm dấu mốc (vai + ranh giới lượt) để mô hình biết "ai đang nói" và "đến lượt mình trả lời". Sai template → mô hình trả lời lạc / loạn vai.
<|im_start|>role …nội dung… <|im_end|> (lặp mỗi tin nhắn) → kết bằng <|im_start|>assistant
💡 Trực giác Như kịch bản sân khấu: mỗi lượt thoại ghi rõ "[VAI]: lời" và dấu hết lượt. Mô hình đọc kịch bản, thấy "[assistant]:" ở cuối còn trống → nó diễn tiếp vai assistant. Token đặc biệt là các dấu mốc đó.
0

Cho sẵn

ví dụ 2 tin nhắn
system = "Bạn là trợ lý."  ·  user = "Chào".   Token đặc biệt: <|im_start|>, <|im_end|> (mỗi cái = 1 token).
1

Gói tin nhắn system

đánh dấu vai
<|im_start|>system Bạn là trợ lý. <|im_end|>
Vì sao: dấu mốc cho mô hình biết đây là chỉ dẫn hệ thống, không phải lời người dùng.
2

Gói user + mở lượt assistant

kết mở
<|im_start|>user Chào <|im_end|> <|im_start|>assistant
Kết bằng <|im_start|>assistant (chưa có <|im_end|>) → mô hình hiểu "tới lượt mình".
3

Đếm token đặc biệt

hội thoại m tin nhắn
Hội thoại có 2 tin nhắn hoàn chỉnh + mở lượt assistant:
<|im_start|> dùng lần ; <|im_end|> dùng lần → tổng token đặc biệt =
Vì sao: token đặc biệt cũng tốn chỗ trong cửa sổ ngữ cảnh; template dài ăn vào ngân sách token.
4

Vì sao đúng template quan trọng

khớp huấn luyện
Phải dùng đúng định dạng mô hình được huấn luyện (mỗi model một kiểu: ChatML, Llama, …) → sai là lệch phân phối, trả lời kém.

2 khung tin nhắn → chuỗi token có mốc

phẳng hóa
Mỗi tin nhắn = 1 start + nội dung + 1 end. Cuối chuỗi mở thêm 1 startassistant (không end) → mô hình "nối lời".
✎ Tự kiểm tra — đáp án
  1. Vì sao cần token đặc biệt? → Đánh dấu vai & ranh giới lượt để mô hình biết ai nói / khi nào tới lượt.
  2. Kết bằng gì? → <|im_start|>assistant (mở lượt assistant đang trống).
