Perplexity — độ "bối rối" của mô hình ngôn ngữ

Họ tên: Ngày: Thời gian: ~12 phút
Vì sao quan trọng. Perplexity = exp của cross-entropy trung bình → "số lựa chọn hiệu dụng" mô hình phân vân giữa. Là chỉ số chuẩn để so sánh mô hình ngôn ngữ: PPL 10 nghĩa là mỗi bước như đoán giữa ~10 token đều khả năng.
PPL = exp( −(1/N) Σ ln p(tokenₜ) ) = exp(cross-entropy trung bình)
💡 Trực giác Nếu mô hình chắc chắn đúng (p=1 cho token thật) → bối rối = 1 (không phân vân). Nếu đoán mò giữa V token đều nhau → PPL ≈ V. PPL = "số mặt xúc xắc đều" tương đương với độ bất định của mô hình.

Độ bất ngờ −ln p từng token

đường = CE trung bình
Cột lam = −ln p mỗi token (token p thấp → cột cao = bất ngờ). Đường cam = CE trung bình. PPL = e^(đường đó).
✎ Tự kiểm tra — đáp án
  1. PPL = 1? → Mô hình hoàn hảo — luôn gán p=1 cho token đúng (không phân vân).
  2. Liên hệ CE? → PPL = exp(cross-entropy trung bình).
Rút ra. PPL = exp(CE trung bình) = số lựa chọn hiệu dụng; thấp là tốt. Bài tiếp (I5): tinh chỉnh rẻ bằng LoRA.
