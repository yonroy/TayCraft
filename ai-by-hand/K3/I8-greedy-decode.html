Greedy Decode — sinh trọn một chuỗi

Họ tên: Ngày: Thời gian: ~11 phút
Vì sao quan trọng. Đây là vòng lặp sinh cơ bản của mọi LLM: token → mô hình → logits → chọn → nối → lặp. Greedy là cách chọn đơn giản nhất (tất định); hiểu nó là hiểu "LLM viết câu thế nào".
lặp: logitsₜ = Model(token₁…tₜ₋₁) → tokenₜ = argmax(logitsₜ) → nối → tới khi <eos>
💡 Trực giác Như viết từng chữ, mỗi chữ chọn "khả dĩ nhất ngay lúc đó" rồi đọc lại cả câu để viết chữ kế. Tất định: cùng đầu vào → cùng đầu ra. Nhanh, nhưng thiển cận (không nhìn xa như beam — Bài G8).

argmax

Vì sao: chọn logit lớn nhất → token đầu của chuỗi.

tới <eos>
argmax( 1, 0, 4 ) = token → chuỗi =
Vì sao: lặp đến khi mô hình phát token kết thúc (<eos>) thì dừng.

dễ lặp

✎ Tự kiểm tra — đáp án
  1. Chọn thế nào? → argmax logits (xác suất cao nhất) mỗi bước.
  2. Token vừa sinh? → Làm đầu vào cho bước kế (tự hồi quy).
Rút ra. Greedy decode = lặp argmax + nối + hồi quy tới <eos>; tất định, dễ lặp. Bài tiếp (I9): giảm lặp bằng repetition penalty.
